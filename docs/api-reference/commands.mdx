---
title: Commands
description: Execute any Redis command via path-based or POST body requests.
---

Any Redis command supported by your local Redis instance can be executed via the REST API.

## Path-based execution

```
GET /{COMMAND}/{arg1}/{arg2}/.../{argN}
```

<ParamField query="_token" type="string">
  API token (alternative to Authorization header)
</ParamField>

### Examples

<CodeGroup>

```bash SET
curl -H "Authorization: Bearer local-dev-token" \
  http://localhost:8000/SET/session/abc123
```

```bash GET
curl -H "Authorization: Bearer local-dev-token" \
  http://localhost:8000/GET/session/abc123
```

```bash HSET
curl -H "Authorization: Bearer local-dev-token" \
  http://localhost:8000/HSET/user:1/name/Alice/email/alice@example.com
```

```bash EXPIRE
curl -H "Authorization: Bearer local-dev-token" \
  http://localhost:8000/SET/cache/data/EX/300
```

```bash DEL
curl -H "Authorization: Bearer local-dev-token" \
  http://localhost:8000/DEL/key1/key2/key3
```

</CodeGroup>

## POST body execution

Send to `/` with a JSON array:

```bash
curl -X POST http://localhost:8000/ \
  -H "Authorization: Bearer local-dev-token" \
  -H "Content-Type: application/json" \
  -d '["SET", "foo", "bar", "EX", 3600]'
```

## POST with body value

For large or binary values, put the value in the request body:

```bash
curl -X POST http://localhost:8000/SET/mykey?EX=3600 \
  -H "Authorization: Bearer local-dev-token" \
  -H "Content-Type: application/json" \
  -d 'my-large-value-here'
```

Equivalent to: `SET mykey my-large-value-here EX 3600`

## Base64 encoding

Request base64-encoded responses:

```bash
curl -H "Authorization: Bearer local-dev-token" \
  -H "Upstash-Encoding: base64" \
  http://localhost:8000/GET/mykey
```

## RESP2 format

Get raw Redis protocol bytes:

```bash
curl -H "Authorization: Bearer local-dev-token" \
  -H "Upstash-Response-Format: resp2" \
  http://localhost:8000/GET/mykey
```

Content-Type: `application/octet-stream`

## Supported command categories

| Category | Supported |
| --- | --- |
| Strings | ✅ |
| Hashes | ✅ |
| Lists | ✅ |
| Sets | ✅ |
| Sorted Sets | ✅ |
| Streams | ✅ |
| Pub/Sub | ✅ (via dedicated endpoints too) |
| Transactions | ✅ (via `/multi-exec`) |
| Scripting | ✅ |
| JSON (Redis Stack) | ✅ with stack profile |
| Blocking commands | ⚠️ Pass through but not useful over REST |

## Error responses

```json
{"error":"ERR wrong number of arguments for 'get' command"}
```

HTTP status: **400**
