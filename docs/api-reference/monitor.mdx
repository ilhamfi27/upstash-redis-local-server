---
title: Monitor
description: Stream all Redis commands in real time via Server-Sent Events.
---

The monitor endpoint streams every command processed by Redis — useful for debugging during development.

**Endpoint:** `POST /monitor`

## Request

```bash
curl -N \
  -H "Authorization: Bearer local-dev-token" \
  -H "Accept: text/event-stream" \
  -X POST http://localhost:8000/monitor
```

## Stream output

```
data: "OK"

data: 1721284005.242090 [0 127.0.0.1:54321] "GET" "mykey"

data: 1721284008.663811 [0 127.0.0.1:54321] "SET" "mykey" "newvalue"
```

## Two-terminal test

**Terminal 1 — start monitor:**

```bash
curl -N -H "Authorization: Bearer local-dev-token" \
  -H "Accept: text/event-stream" \
  -X POST http://localhost:8000/monitor
```

**Terminal 2 — run commands:**

```bash
curl -H "Authorization: Bearer local-dev-token" \
  http://localhost:8000/SET/debug/key

curl -H "Authorization: Bearer local-dev-token" \
  http://localhost:8000/GET/debug/key
```

Terminal 1 shows each command as it executes.

## Notes

- Uses Redis `MONITOR` command under the hood
- Monitor has performance overhead — use only in development
- Stream runs until the client disconnects
- No cloud quota consumed
