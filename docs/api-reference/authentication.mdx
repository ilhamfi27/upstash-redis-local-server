---
title: Authentication
description: API tokens, read-only access, and ACL RESTTOKEN support.
---

All endpoints require authentication except `/health`, `/ready`, and `/dashboard/api/stats`.

## Bearer token

```bash
curl -H "Authorization: Bearer local-dev-token" \
  http://localhost:8000/PING
```

## Query parameter

Useful for browser testing:

```
http://localhost:8000/PING?_token=local-dev-token
http://localhost:8000/GET/mykey?_token=local-dev-token
```

## Token types

| Token | Default value | Permissions |
| --- | --- | --- |
| Full-access | `local-dev-token` | All Redis commands |
| Read-only | `local-readonly-token` | Read commands only |

Configure via flags or environment:

```bash
--token local-dev-token
--readonly-token local-readonly-token

# or
UPSTASH_TOKEN=local-dev-token
UPSTASH_READONLY_TOKEN=local-readonly-token
```

## Read-only restrictions

Read-only tokens block:

**Write commands:** `SET`, `DEL`, `INCR`, `HSET`, `LPUSH`, `SADD`, `ZADD`, `PUBLISH`, etc.

**Powerful read commands:** `KEYS`, `SCAN`, `FLUSHALL`, `FLUSHDB`, `CONFIG`, `SHUTDOWN`

Example — write blocked:

```bash
curl -H "Authorization: Bearer local-readonly-token" \
  http://localhost:8000/SET/key/value
# {"error":"NOPERM this user has no permissions to run the 'SET' command"}
```

Example — read allowed:

```bash
curl -H "Authorization: Bearer local-readonly-token" \
  http://localhost:8000/GET/key
# {"result":"value"}
```

## ACL RESTTOKEN

Generate per-user REST tokens (mirrors Upstash Cloud behavior):

```bash
curl -X POST http://localhost:8000/ \
  -H "Authorization: Bearer local-dev-token" \
  -H "Content-Type: application/json" \
  -d '["ACL", "RESTTOKEN", "default", "your-redis-password"]'
```

Response:

```json
{ "result": "generated-token-string" }
```

Use the returned token as your Bearer token. Commands run with that Redis user's permissions.

<Warning>
  ACL RESTTOKEN requires Redis ACL users configured. Default Redis has no password — use full-access token for standard local dev.
</Warning>

## Unauthorized response

```json
{ "error": "Unauthorised" }
```

HTTP status: **401**
