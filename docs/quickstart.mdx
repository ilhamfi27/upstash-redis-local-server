The fastest way to get Upstash Redis Local running is the `start.sh` script bundled in the repository. It detects your Docker environment, picks the right Compose profile, and starts both Redis and the local server for you — no manual configuration required.

## Prerequisites

You need [Docker](https://docs.docker.com/get-docker/) installed and the Docker daemon running before you begin. The quick-start script handles everything else.

<Steps>

### Clone the repository

```bash
git clone https://github.com/UpstashOrg/upstash-redis-local.git
cd upstash-redis-local
```

### Start the server

Run the quick-start script. It inspects your Docker setup, selects the appropriate Compose profile, and brings up both the Redis container and the Upstash Redis Local server.

```bash
./start.sh
```

The script prints the selected profile and the URLs the server is listening on once everything is healthy. Both services start in detached mode, so your terminal is free immediately after.

<Tip>
  If you prefer to watch the startup logs in real time, run `docker-compose logs -f upstash-local` after the script exits.
</Tip>

### Verify the server is running

Send a `PING` command through the REST API to confirm the server is accepting requests.

```bash
curl "http://localhost:8000/PING?_token=local-dev-token"
```

You should receive:

```json
{"result":"PONG"}
```

If you see that response, the server is up and correctly connected to Redis.

### Open the dashboard

Navigate to [http://localhost:8000/dashboard](http://localhost:8000/dashboard) in your browser. The built-in dashboard lets you browse keys, inspect values, and run ad-hoc commands against your local Redis instance — no extra tooling needed.

### Connect your application

Use the `@upstash/redis` SDK to connect to the local server. The only change from your production configuration is the URL and token.

```typescript
import { Redis } from '@upstash/redis'

const redis = new Redis({
  url: 'http://localhost:8000',
  token: 'local-dev-token',
})

await redis.set('foo', 'bar')
const value = await redis.get('foo') // 'bar'
console.log(value)
```

Install the SDK if you haven't already:

```bash
npm install @upstash/redis
```

</Steps>

## Available endpoints

Once the server is running, the following URLs are available on `http://localhost:8000`:

| Path | Description |
|---|---|
| `/<COMMAND>` | Execute any Redis command via REST |
| `/pipeline` | Send a batch of commands in a single request |
| `/multi-exec` | Execute commands inside a Redis transaction |
| `/publish` | Publish a message to a pub/sub channel |
| `/subscribe` | Subscribe to a pub/sub channel |
| `/monitor` | Stream all commands passing through Redis |
| `/dashboard` | Web-based key browser and command runner |
| `/health` | Health-check endpoint — returns `200 OK` when ready |

## Default credentials

| Token type | Value |
|---|---|
| Read-write token | `local-dev-token` |
| Read-only token | `local-readonly-token` |

<Warning>
  These tokens are hardcoded defaults meant for local development only. Never use them in a production environment or commit them to a public repository.
</Warning>

## Switching back to the Upstash cloud

When you're ready to deploy, point your application back at the real Upstash service by updating your environment variables. No code changes are required.

```bash
# Local development
UPSTASH_REDIS_REST_URL=http://localhost:8000
UPSTASH_REDIS_REST_TOKEN=local-dev-token

# Production (Upstash cloud)
UPSTASH_REDIS_REST_URL=https://<your-endpoint>.upstash.io
UPSTASH_REDIS_REST_TOKEN=<your-upstash-token>
```

<Info>
  The `@upstash/redis` SDK reads `UPSTASH_REDIS_REST_URL` and `UPSTASH_REDIS_REST_TOKEN` automatically when you use `Redis.fromEnv()`, making environment-based switching seamless.
</Info>
