Upstash Redis Local is a lightweight local server that speaks the exact same REST API as Upstash Redis in the cloud. Point your `@upstash/redis` SDK — or any HTTP client — at `http://localhost:8000` and get **unlimited requests**, no daily quotas, and no billing surprises while you build.

## Why use Upstash Redis Local?

The Upstash free tier limits you to 10,000 requests per day. During active development you can blow through that in minutes. Upstash Redis Local removes that ceiling entirely — every call hits a real Redis database on your machine.

## Get started in 3 steps

<Steps>
  <Step title="Clone and start the server">
    ```bash
    git clone https://github.com/aine1100/Upstash-Redis-Local-server.git
    cd Upstash-Redis-Local-server
    ./start.sh
    ```
    The script detects Docker, picks the right profile, and launches the server on port 8000.
  </Step>
  <Step title="Verify it's running">
    ```bash
    curl "http://localhost:8000/PING?_token=local-dev-token"
    # → {"result":"PONG"}
    ```
  </Step>
  <Step title="Point your app at the local server">
    ```typescript
    import { Redis } from '@upstash/redis'

    const redis = new Redis({
      url: 'http://localhost:8000',
      token: 'local-dev-token',
    })

    await redis.set('hello', 'world')
    console.log(await redis.get('hello')) // 'world'
    ```
    No other code changes needed — the SDK is fully compatible.
  </Step>
</Steps>

<Tip>
  Open the **dashboard** at [http://localhost:8000/dashboard](http://localhost:8000/dashboard) to browse your keys, track request counts, and see how much cloud quota you've saved.
</Tip>
