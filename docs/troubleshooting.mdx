## localhost:8000 connection refused

**Cause:** The proxy container isn't running.

**Fix:**

```bash
docker ps   # check for upstash-local on port 8000
./start.sh  # restart
```

If container shows `Created` but not `Up`, check logs:

```bash
docker logs upstash-redis-local-upstash-local-1
```

---

## Port 6379 already allocated

**Cause:** Another Redis container (e.g. `redis-local`) is using port 6379.

**Fix — option A:** Use external profile:

```bash
docker compose --profile external up -d upstash-local-external
```

**Fix — option B:** Stop conflicting container:

```bash
docker stop redis-local
docker compose up -d
```

`start.sh` auto-detects this and switches to external profile.

---

## Redis connection failed on startup

**Cause:** Proxy can't reach Redis.

**Fix:**

| Environment | Redis address |
| --- | --- |
| Docker Compose (default) | `redis:6379` |
| Docker → host Redis | `host.docker.internal:6379` |
| Native binary | `localhost:6379` |
| Git Bash on Windows | `localhost:6379` (not `:6379` alone) |

Restart proxy after Redis is up:

```bash
docker restart upstash-redis-local-upstash-local-external-1
```

## Unauthorized / 401 errors

**Cause:** Missing or wrong token.

**Fix:**

```bash
# Default Docker Compose token
curl -H "Authorization: Bearer local-dev-token" http://localhost:8000/PING

# Or in browser
http://localhost:8000/PING?_token=local-dev-token
```

---

## curl JSON errors on Windows PowerShell

**Cause:** PowerShell's `curl` is `Invoke-WebRequest` — it mangles JSON bodies.

**Fix:** Use Git Bash, or `curl.exe` for simple GETs:

```powershell
curl.exe -s http://localhost:8000/health
curl.exe -s -H "Authorization: Bearer local-dev-token" http://localhost:8000/PING
```

For POST with JSON, always use **Git Bash**.

---

## Container unhealthy

**Cause:** Redis not reachable or server crashed.

```bash
docker logs <container-name>
curl http://localhost:8000/health
```

---

## Empty dashboard / no keys

**Cause:** No data seeded yet, or wrong token in key browser.

**Fix:**

```bash
curl -H "Authorization: Bearer local-dev-token" \
  http://localhost:8000/SET/testkey/hello

# Or seed via CLI
./bin/upstash-local seed --keys 10 --prefix dev:
```

In dashboard, use token `local-dev-token` and pattern `*`.

---

## @upstash/redis still hitting cloud

**Cause:** App still using cloud env vars.

**Fix:**

```bash
./bin/upstash-local use dev
./bin/upstash-local status
```

Verify your app loads `.env.local` and not production `.env`.
