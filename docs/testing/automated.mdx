## Unit tests

Requires Go 1.22+:

```bash
cd upstash-redis-local
go test -v ./internal/
```

Or via Make:

```bash
make test
```

### Without Go installed (Docker)

```bash
docker run --rm -v "$(pwd):/app" -w /app golang:1.22-alpine \
  sh -c "go mod tidy && go test -v ./internal/"
```

### Test coverage

| Test | Verifies |
| --- | --- |
| `TestHealth` | `/health` returns 200 |
| `TestPing` | Authenticated PING works |
| `TestSetGet` | Path-based SET/GET |
| `TestPipeline` | POST `/pipeline` |
| `TestMultiExec` | POST `/multi-exec` |
| `TestReadOnlyToken` | Write blocked for readonly |
| `TestUnauthorized` | 401 without token |
| `TestDashboardStats` | Metrics tracking |

## CI pipeline

GitHub Actions runs on every push:

**`.github/workflows/ci.yml`** — build, test, vet, Docker build

**`.github/workflows/integration.yml`** — full integration against real Redis:

```yaml
- docker run -d --name redis-test -p 6379:6379 redis:7-alpine
- go build -o bin/upstash-redis-local .
- ./bin/upstash-redis-local --redis localhost:6379 --token local-dev-token &
- curl -sf http://localhost:8000/health
- curl -sf -H "Authorization: Bearer local-dev-token" http://localhost:8000/PING
- curl -sf -X POST http://localhost:8000/pipeline ...
- curl -sf -X POST http://localhost:8000/multi-exec ...
```

## Add to your project's CI

```yaml
name: Tests
on: [push, pull_request]

jobs:
  test:
    runs-on: ubuntu-latest
    services:
      redis:
        image: redis:7-alpine
        ports: ['6379:6379']

    steps:
      - uses: actions/checkout@v4

      - name: Start Upstash Redis Local
        run: |
          git clone https://github.com/aine1100/Upstash-Redis-Local-server.git /tmp/upstash-local
          cd /tmp/upstash-local/upstash-redis-local
          go build -o server .
          ./server --redis localhost:6379 --token local-dev-token &
          sleep 2
          curl -sf http://localhost:8000/health

      - name: Run your tests
        run: npm test
        env:
          UPSTASH_REDIS_REST_URL: http://localhost:8000
          UPSTASH_REDIS_REST_TOKEN: local-dev-token
```

<Tip>
  Running Upstash Redis Local in CI means **zero cloud quota** consumed by your test pipeline.
</Tip>
