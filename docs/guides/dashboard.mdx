The built-in dashboard gives you visibility into local usage — and shows how much Upstash Cloud quota you've saved.

## Open the dashboard

```
http://localhost:8000/dashboard
```

No auth required for the UI. Key browsing requires your API token.

## What you'll see

<CardGroup cols={2}>
  <Card title="Total Requests" icon="hashtag">
    Every authenticated API call is counted.
  </Card>
  <Card title="Cloud Quota Saved" icon="shield-check">
    Same count — requests that would have hit Upstash Cloud.
  </Card>
  <Card title="Free Tier Limit" icon="gauge">
    Reference line at 10,000/day (Upstash free tier).
  </Card>
  <Card title="Uptime" icon="clock">

## Key browser

1. Enter a key pattern (default: `*`)
2. Enter your token (`local-dev-token`)
3. Click **Browse Keys**

Results show key name, type, TTL, and a preview of the value.

## Stats API

Programmatic access to metrics:

```bash
curl http://localhost:8000/dashboard/api/stats
```

Response:

```json
{
  "total_requests": 142,
  "uptime_seconds": 3600,
  "commands": { "GET": 80, "SET": 40, "PING": 22 },
  "free_tier_quota": 10000,
  "quota_saved": 142,
  "cloud_quota_remaining": 9858,
  "unlimited_local": true
}
```

## Keys API

```bash
curl "http://localhost:8000/dashboard/api/keys?pattern=dev:*&_token=local-dev-token"
```

Returns all matching keys with type, value, and TTL.

## Use cases

- **During dev** — watch request count climb during hot reload; see why cloud quota disappears fast
- **After test runs** — confirm how many commands your suite executes
- **Debugging** — browse keys without Redis CLI or Redis Insight
- **Demos** — show stakeholders unlimited local dev vs cloud limits
