title: SDK Integration

Upstash Redis Local is fully compatible with the official [`@upstash/redis`](https://github.com/upstash/upstash-redis) SDK and any HTTP client.

## TypeScript / Node.js

```bash
npm install @upstash/redis
```

```typescript
import { Redis } from '@upstash/redis'

const redis = new Redis({
  url: process.env.UPSTASH_REDIS_REST_URL ?? 'http://localhost:8000',
  token: process.env.UPSTASH_REDIS_REST_TOKEN ?? 'local-dev-token',
})

// Strings
await redis.set('user:1', { name: 'Alice', age: 30 })
const user = await redis.get('user:1')

// Pipeline
const pipe = redis.pipeline()
pipe.set('key1', 'a')
pipe.set('key2', 'b')
pipe.get('key1')
const results = await pipe.exec()

// Hashes, lists, sets — all work
await redis.hset('profile:1', { email: 'a@b.com' })
await redis.lpush('queue', 'job1', 'job2')
await redis.sadd('tags', 'redis', 'local')
```

## Next.js

```typescript
// lib/redis.ts
import { Redis } from '@upstash/redis'

export const redis = new Redis({
  url: process.env.UPSTASH_REDIS_REST_URL!,
  token: process.env.UPSTASH_REDIS_REST_TOKEN!,
})
```

```bash
# .env.local
UPSTASH_REDIS_REST_URL=http://localhost:8000
UPSTASH_REDIS_REST_TOKEN=local-dev-token
```

## Cloudflare Workers

```typescript
import { Redis } from '@upstash/redis'

export default {
  async fetch(request, env) {
    const redis = new Redis({
      url: 'http://localhost:8000',  // use tunnel for remote dev
      token: 'local-dev-token',
    })
    await redis.set('edge-key', 'edge-value')
    return new Response(await redis.get('edge-key'))
  },
}
```

<Tip>
  For Cloudflare Workers local dev, use `wrangler dev` with a tunnel to your local server, or point at cloud Upstash for worker-specific testing.
</Tip>

## Python

```python
from upstash_redis import Redis

redis = Redis(
    url="http://localhost:8000",
    token="local-dev-token",
)

redis.set("key", "value")
print(redis.get("key"))
```

## Raw HTTP (any language)

```bash
# Path-based command
curl -H "Authorization: Bearer local-dev-token" \
  http://localhost:8000/SET/foo/bar

# POST body (SDK style)
curl -X POST http://localhost:8000/ \
  -H "Authorization: Bearer local-dev-token" \
  -H "Content-Type: application/json" \
  -d '["SET", "foo", "bar", "EX", 3600]'
```

## Read-only token for frontend

Use the read-only token when exposing credentials in browser code:

```typescript
const redis = new Redis({
  url: 'http://localhost:8000',
  token: 'local-readonly-token',  // blocks writes + KEYS/SCAN
})
```

See [Authentication](/api-reference/authentication).
